# 人物志：性格测试逻辑规范 v1

> 当前阶段只确定测量与匹配逻辑。交互、配色和版式暂不作为产品决策依据。

## 1. 产品边界

这是一个有明确计分依据的娱乐型人格探索工具，不是临床心理量表，也不声称能还原历史人物的真实人格。

结果中的历史人物是基于文学、史书和大众文化形成的“叙事原型”。程序真正测量的是用户在五个连续维度上的选择倾向，然后寻找轮廓相近的原型。

必须遵守四条原则：

1. 不把某道题直接绑定到某个人物。
2. 不把人物预设分数伪装成用户实测分数。
3. 高分、低分都描述为倾向，不暗示道德优劣。
4. 匹配不清晰时允许输出双原型，不能硬造“精准度”。

## 2. 底层人格模型

采用五因素人格模型的结构，但使用更容易理解的中文名称。

| 代码 | 展示名称 | 对应概念 | 高分倾向 | 低分倾向 |
| --- | --- | --- | --- | --- |
| O | 探索开放 | Openness | 偏爱新思路、抽象信息和变化 | 偏爱熟悉方法、具体信息和可预期性 |
| C | 结构执行 | Conscientiousness | 计划、持续、按标准完成 | 灵活、即兴、根据现场调整 |
| E | 外向驱动 | Extraversion | 从互动中获得能量，主动表达和发起 | 从独处中恢复能量，先观察再表达 |
| A | 协作共情 | Agreeableness | 关注关系、信任与共同利益 | 关注原则、竞争与独立判断 |
| R | 复原稳定 | Emotional stability | 压力下平稳，恢复速度较快 | 对风险更敏感，情绪信号更强 |

这里的“低分倾向”不是缺点。例如低 E 可以意味着深度专注，低 A 可以意味着敢于质疑，低 R 可以意味着对风险和细节更加警觉。

### 为什么不用原图中的五个维度

原图的“判断力、行动力、边界感、关系感知、稳定性”存在明显重叠：

- 判断力和行动力会被具体能力、经验和场景强烈影响，不是稳定人格特质。
- 边界感与关系感知并非相互独立。
- 五个指标缺少成熟的理论结构，难以设计均衡题目。

新模型保留用户能感受到的行为差异，同时减少维度间重复。

## 3. 测试长度与流程

测试由两段组成：

1. **25道核心情境题**：每个维度5题。
2. **0–3道自适应辨析题**：只有当前两个人物候选过于接近时才出现。

因此总题数为25–28题，预计完成时间5–7分钟。

核心题回答完成后，程序先计算临时五维分数和前两名人物：

- 如果第一、第二候选的标准化距离差 `D2 - D1 >= 0.035`，直接生成结果。
- 如果差值小于 `0.035`，从两位候选差异最大的维度中抽取辨析题。
- 最多追加3题。
- 追加后仍无法拉开差距，则输出“双原型”，例如“诸葛亮 × 张良”，并明确说明两种策略在用户身上同时存在。

`0.035` 是首版工程阈值，正式发布前应通过模拟分布和小规模用户测试校准。

## 4. 题目设计

### 4.1 基本结构

每题描述一个日常可想象的场景，提供四个都说得通的选择。四个选项对应：

```text
-3  -1  +1  +3
```

每题只计入一个主维度。这样做牺牲少量“情境复杂度”，换取可解释性，避免一项回答同时改变多个不相关维度。

例题（结构执行 C）：

> 你接到一个两周后交付、过程完全由你安排的任务。

- 我通常先动手，边做边决定后面的步骤。`-3`
- 先确定最关键的一两步，其余保持弹性。`-1`
- 会列出主要节点，并定期检查进度。`+1`
- 会拆到具体步骤和时间，尽量按计划推进。`+3`

四个选项分别体现即兴适应与结构规划的不同收益，没有“正确答案”。

### 4.2 题库均衡

每个维度的5道核心题必须跨越不同生活领域：

- 工作与学习
- 合作与关系
- 冲突与压力
- 新环境与不确定性
- 个人恢复与长期选择

每个维度至少有2道反向呈现题，避免用户发现“越靠后分越高”。选项在页面上的视觉顺序与实际分值顺序不固定，但同一用户重复测试时保持确定，便于复测。

### 4.3 避免社会赞许偏差

题目不能使用“负责/不负责”“理性/冲动”“善良/自私”之类带有明显价值判断的措辞。每个选项同时包含一种收益和一种代价。

不使用以下题型：

- “你是否总能冷静处理问题？”
- “朋友需要帮助时你会不会伸出援手？”
- “你是不是一个有责任心的人？”

这类题主要测量用户想如何评价自己。

### 4.4 回答稳定度

每个维度的核心题中设置两组语义镜像题。镜像题不会复用文案和场景，但测量同一取向。

把镜像题转换到相同计分方向后，维度回答稳定度为：

```text
C_d = 1 - mean(|r_i - r_j|) / 6
```

其中 `r_i`、`r_j` 是两组镜像题的回答，取值范围为 `-3, -1, +1, +3`。

该数值只用于：

- 判断哪些维度需要辨析题；
- 调整人物距离计算中的维度权重；
- 在结果中提示“你的选择在不同场景下更灵活”。

不能把它显示成“测试准确率”。

## 5. 五维计分

对维度 `d`，取该维度所有已回答题目的平均值：

```text
M_d = mean(r_1, r_2, ... r_n)
```

将 `[-3, +3]` 映射到 `[10, 90]`：

```text
S_d = round(50 + 40 × M_d / 3)
```

选择10–90而不是0–100，原因是短测验没有足够信息支持“绝对0分”或“绝对100分”。

展示时必须称为“倾向分”或“位置”，不能称为人群百分位。没有真实常模数据前，不显示“超过了84%的人”。

### 示例

某用户在探索开放维度的5个核心题中得到：

```text
+3, +1, +1, -1, +3
```

则：

```text
M_O = 1.4
S_O = round(50 + 40 × 1.4 / 3) = 69
```

结果为“探索开放 69”，含义是当前回答更偏向开放探索端，不是69%的准确率。

## 6. 57位人物原型

每个人物包含：

```js
{
  id: "zhuge-liang",
  name: "诸葛亮",
  era: "三国·蜀汉",
  archetype: "长期主义的系统规划者",
  vector: { O: 72, C: 88, E: 42, A: 68, R: 79 },
  tags: ["系统思考", "责任驱动", "长期布局"],
  narrativeBasis: "基于大众文化中的军师与治政者形象",
}
```

人物向量只决定匹配位置。用户的维度分必须始终来自用户回答，不能在匹配人物后被人物向量覆盖。

### 6.1 人物匹配距离

用户向量为 `S`，人物向量为 `P`。首先根据回答稳定度给每个维度一个温和权重：

```text
W_d = 0.8 + 0.2 × C_d
```

所以任何维度的权重都不会低于0.8，避免程序因为回答不一致而“忽略”某个维度。

标准化距离为：

```text
D(P) = sqrt(
  sum(W_d × ((S_d - P_d) / 80)²)
  /
  sum(W_d)
)
```

按距离从小到大排列57个人物。距离最小的是主候选。

界面可以把距离转换为便于比较的“轮廓接近度”：

```text
Similarity(P) = round(100 × (1 - D(P)))
```

它只能用于候选之间的相对比较，必须在说明中注明“不是心理测量准确率”。

### 6.2 双原型

完成最多3道辨析题后，如果：

```text
D2 - D1 < 0.01
```

则输出双原型。报告以用户的五维实测分为主体，分别解释两个原型与用户相似的部分，不能把两套固定文案简单拼接。

## 7. 自适应辨析题

每个维度准备至少3道不在核心题库中的辨析题。

需要辨析时：

1. 找出前两位人物向量差值最大的维度；
2. 优先选择该维度中用户尚未回答、且与已答场景领域不同的题；
3. 重新计算该维度平均分和人物排名；
4. 若差距仍不足，再选择下一个区分度最高的维度；
5. 最多3题。

辨析题不会获得更高权重，也不能直接绑定人物。

## 8. 报告生成

结果报告由用户分数动态生成，不使用“诸葛亮固定得分88/82/73”之类模板。

报告包含六部分：

1. **五维轮廓**：展示用户实际分数。
2. **主原型/双原型**：展示最近人物以及叙事依据。
3. **为什么匹配**：指出用户分数与人物向量最接近的2–3个维度。
4. **回答证据**：引用3个真正影响结果的场景选择。
5. **优势与过度使用风险**：每个倾向同时写收益和代价。
6. **邻近人物**：显示第二至第四候选及接近度。

报告文本由“维度解释规则 + 用户回答证据 + 人物元数据”组合生成。人物可以有专属标题和简介，但不能拥有一套与用户答案无关的固定心理结论。

## 9. 匹配清晰度

不显示“准确度84%”。结果可以显示“匹配清晰度”，计算依据包括：

- 第一与第二人物的距离差；
- 五个维度的回答稳定度；
- 是否触发以及完成辨析题。

首版公式：

```text
Gap = clamp((D2 - D1) / 0.10, 0, 1)
Consistency = mean(C_O, C_C, C_E, C_A, C_R)
Clarity = round(100 × (0.65 × Gap + 0.35 × Consistency))
```

展示文案采用区间，不突出精确数字：

- `75–100`：轮廓较清晰
- `50–74`：存在邻近原型
- `< 50`：情境型/混合轮廓

这个指标描述“本次结果是否容易区分”，不是描述用户是否认真，也不是心理准确率。

## 10. 质量校验

### 10.1 发布前程序校验

人物库完成后运行自动模拟：

1. 至少生成200,000组五维向量。
2. 确认57个人物全部存在可匹配区域。
3. 任意两个人物向量不能完全相同。
4. 检查是否有少数中心人物吸收绝大多数结果。
5. 用极端向量、全中间向量和随机向量测试边界。
6. 同一答案序列必须得到完全相同的结果。
7. 只修改一道题时，只有对应主维度发生变化。

如果某个人物在模拟中永远无法成为第一候选，必须调整或删除，不能把它计入可匹配人物数量。

### 10.2 有真实样本后的测量校验

没有用户样本前，只能称为“结构化娱乐测试”。收集明确同意的匿名样本后再评估：

- 每个维度的内部一致性；
- 2–4周后的重测相关；
- 题目是否存在明显性别、年龄或职业偏差；
- 各选项是否极端冷门；
- 人物结果分布是否异常集中。

在完成这些验证前，不使用“专业心理测试”“精准测量”等营销表述。

## 11. 首版验收标准

- 25道核心题严格按每维5题分配。
- 每个维度至少2道反向呈现题、2组镜像题。
- 57个人物全部通过可达性模拟。
- 用户分数在匹配前生成，匹配后不得修改。
- 接近结果会触发0–3道辨析题。
- 无法清晰区分时能输出双原型。
- 报告至少引用3条真实回答证据。
- 同一答案始终得到同一结果。
- 分数、距离和清晰度公式在产品内可查看。

## 12. 尚待确认的内容

逻辑进入题库编写前，需要锁定以下产品选择：

1. 历史人物范围采用“中国历史人物”还是“世界历史人物”。
2. 人物依据更偏正史公众形象，还是允许文学形象进入。
3. 测试语境更偏职场、日常生活，还是二者均衡。

这三项只影响人物库和题目措辞，不改变上述测量与匹配公式。

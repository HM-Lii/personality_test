**关键学习**：
1. **加人降集中度的正确姿势是"紧贴霸主"**（距离 0.07-0.08，胞腔被霸主夹小），而非"落在霸主与中部之间"（那样新人独占新空腔，成新霸主）。
2. **打地鼠效应**：每调一个向量位置，就让出一片空腔被另一人物独占。需要把 Top 区人物向量同时往极端推，才能整体压缩胞腔。
3. **sd 非根因**：sd=14 实验反而更差（张飞 23.76%，低命中 23 人）。sd 降低让中部人物密集竞争，极端区域用户占比相对升。

## 5. 结构性限制：集中度 ↔ 辨析率矛盾

### 根因

- `needsCalibration`：前两名 gap < 阈值时触发辨析
- 集中度低要求人物密集（胞腔小且均匀，~7%），用户因此大量落在胞腔边界附近，前两名 gap 小
- 辨析率低要求人物稀疏（胞腔大，用户在中心，gap 大）
- **两者在硬最近邻算法下不可兼得**

### 证据

- baseline 57 人：Top1 17.90%，辨析 60.98%
- 62 人密集：Top1 7.51%，辨析 74.46%（threshold 0.035）
- threshold 0.025：辨析压回 60.93%（≈baseline）
- 无论 57/61/62 人，辨析率都 >60%

### Phase B 建议

要同时满足集中度 <8% 且辨析 <50%，需改 `rankFigures` 算法（超出阶段 A "零样本理论优化"范围）：

1. **softmax 概率分配**：距离越近概率越高，非硬最近邻。大胞腔人物概率被邻居分摊。
2. **绝对距离阈值辨析**：用 `ranking[0].distance < 阈值` 替代相对 gap，避免胞腔小导致 gap 普遍小。
3. **分维度匹配 + 加权**：按维度分别匹配，减少多维距离中和导致的并列。
